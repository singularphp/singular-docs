Funcionalidade: Cadastro de clientes#

Como usuário com perfil (1. Diretor, 2. Gerente) no SisGeop, eu quero gerenciar os clientes da JTEL no SisGeop.

Modelo de dados#

Tabelas relacionadas#

Tabela Descrição
cliente Armazena os registros de clientes do sistema
contato Armazena os registros de contatos dos clientes
regional Armazena os registros de regionais dos clientes

Listar Clientes#

Contexto

  • Para acessar essa funcionalidade o usuário acessa o menu Cadastrar > Clientes;
  • O sistema deverá exibir a lista de todos os registros cadastrados quando o usuário entrar na tela;

PROTÓTIPO#

1. LISTA DE CLIENTES#

QUANDO o usuário acessar a tela de listagem de clientes

ENTÃO o sistema exibirá os todos os usuários cadastrados no sistema em uma tabela com as colunas definidas no protótipo seguindo a mesma ordem de exibição.

2. FILTRO DE CLIENTES#

QUANDO o usuário preencher os campos do formulário Filtro e clicar no botão Pesquisar

ENTÃO o sistema irá aplicar o filtro desejado na lista de clientes, retornando apenas clientes que combinam com o grupo de filtros aplicado

Regras

O formulário Filtro pode ser expandido ou encolhido sempre ao clicar na barra de títulos do formulário. E o status encolhido/expandido deverá ficar salvo na sessão do usuário na máquina, mantendo uma memória sempre que o usuário utilizar o SisGeop na mesma máquina.

3. LIMPAR FILTRO#

QUANDO o usuário clicar no botão Limpar

ENTÃO o sistema limpa o formulário Filtro e atualiza a lista de clientes

4. APLICAR FILTRO#

QUANDO o usuário clicar no botão Pesquisar

ENTÃO o sistema aplica o filtro preenchido pelo usuário no formulário Filtro e atualiza a lista de clientes

5. TOTALIZAÇÃO DE REGISTROS#

QUANDO o usuário acessar a lista de clientes

ENTÃO o sistema apresenta o resultado da consulta

E a totalização de registros com a mensagem Mostrando x de y registros

6. PAGINAÇÃO#

QUANDO o usuário acessar a lista de clientes

ENTÃO o sistema apresenta o resultado da consulta

E um componente de paginação que possibilita navegar entre as páginas dos resultados

REGRAS devem ser exibidos em cada página, 25 registros

7. EDITAR CLIENTE#

QUANDO o usuário clicar na opção Editar

ENTÃO o sistema aciona o caso Editar Cliente

8. EXCLUIR CLIENTE#

QUANDO o usuário clicar na opção Excluir

ENTÃO o sistema aciona o caso Excluir Cliente

9. NOVO CLIENTE#

QUANDO o usuário clicar no botão Novo Cliente

ENTÃO o sistema aciona o caso Incluir Cliente

Incluir Cliente#

Contexto

  • Para acessar essa funcionalidade o usuário aciona o botão Novo cliente a partir da tela de listagem de clientes;
  • O sistema deverá exibir o formulário para a criação de um novo cliente;

PROTÓTIPO#

1. FORMULÁRIO DE INCLUSÃO DE CLIENTE#

QUANDO o usuário acionao o botão Novo Cliente a partir da tela de listagem de clientes

ENTÃO o sistema exibe o formulário de inclusão de cliente

2. CANCELAR INCLUSÃO DE CLIENTE#

QUANDO o usuário clicar no botão Cancelar

ENTÃO o sistema sai da tela de inclusão de cliente e retorna para a tela de listagem de clientes

3. SALVAR CLIENTE#

QUANDO o usuário clicar no botão Salvar

ENTÃO o sistema valida o preenchimento do formulário

E Salva os dados preenchidos no formulário criando um novo registro de cliente no banco de dados

E Exibe a mensagem "Registro incluído com sucesso" e redirecionando o usuário para a edição do cliente criado

3.1 Formulário inválido#

CASO o formulário não esteja devidamente preenchido

ENTÃO o sistema apresenta a seguinte mensagem "Os campos destacados são de preenchimento obrigatório"

E destaca os campos que precisam ser preenchidos para o usuário

3.2 Razão social duplicada#

CASO a razão social informada no registro que o usuário estiver inserindo já esteja em utilização por outro cliente

ENTÃO o sistema exibirá a seguinte mensagem "Já existe um cliente com esta razão social"

E interromperá a inserção do registro do cliente.

4. TROCAR LOGOMARCA#

QUANDO o usuário clicar no botão Trocar logomarca

ENTÃO o sistema exibe janela para seleção de um arquivo de imagem na máquina do usuário

4.1 Formato inválido#

CASO o usuário selecione um arquivo que não seja um formato de imagem válido (png, jpg, gif, bmp)

ENTÃO o sistema exibe a mensagem __"Formato de arquivo inválido. As extensões permitidas são (png, jpg, gif e bmp)

4.2 Formáto válido#

CASO o usuário selecione um arquivo que seja um formato de imagem válido

ENTÃO o sistema atualiza a logomarca do cliente na tela de inclusão de cliente

OBRIGATORIEDADE DE CAMPOS#

O sistema deverá exibir de forma destacada os campos do formulário que são de preenchimento obrigatório

Editar Cliente#

Contexto

  • O sistema deverá exibir o formulário para a edição de um cliente carregando todos os dados cadastrados para o cliente nos respectivos campos de edição do formulário;

QUANDO o usuario clicar na aba Regionais

ENTÃO o sistema deverá carregar uma lista com as regionais do cliente

6. FORMULÁRIO DE INCLUSÃO DE REGIONAIS#

QUANDO o usuário clicar no botão Nova regional

ENTÃO o sistema irá exibir o formulário de inclusão de regional

7. INCLUIR REGIONAL#

QUANDO o usuário clicar no botão Incluir

ENTÃO o sistema irá validar o formulário de inclusão de regional

ENTÃO o sistema exibe o formulário de inclusão de regional

E altera o botão Incluir para Alterar

E carrega os dados da regional editada no formulário de alteração de regional

9.1 Confirmar exclusão#

ENTÃO o sistema realiza a exclusão lógica do regiostro da regional

E exibe a mensagem "Registro excluído com sucesso"

9.2 Cancelar exclusão#

CASO o usuário clique no botão Não

ENTÃO o sistema cancela a exclusão do registro da regional e fecha a mensagem de confirmação

10. ABA DE CONTATOS#

Excluir Cliente#

Contexto

  • Para acessar essa funcionalidade o usuário aciona o botão Excluir a partir da tela de listagem de clientes;
  • A exclusão de um usuário deverá ser uma exclusão Lógica de forma que o registro da despesa nunca será fisicamente excluído do banco de dados;

QUANDO o usuário acionar o botão Excluir em uma linha na lista de clientes

ENTÃO o sistema exibe uma janela de confirmação com a mensagem "Deseja realmente excluir o registro selecionado?"

QUANDO o usuário confirmar a exclusão do registro

ENTÃO o sistema efetua a desativação Lógica do registro do cliente

E atualiza a lista de clientes de acordo com o filtro previamente definido pelo usuário
